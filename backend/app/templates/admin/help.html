1. ภาพรวมระบบ

BK-Moph Notify คือระบบแจ้งเตือนผลวิกฤต (Critical Alert) สำหรับโรงพยาบาล โดยดึงข้อมูลจากฐานข้อมูล HOSxP แล้วส่งข้อความแจ้งเตือนผ่าน LINE ไปยังกลุ่มเจ้าหน้าที่ที่เกี่ยวข้อง เช่น ผลแล็บวิกฤต (Lab Critical) หรือยาวิกฤต (Drug Critical)

เจ้าหน้าที่รับทราบข้อมูลแล้วสามารถ กดรับเคส (Claim) ผ่านปุ่มในข้อความ LINE ได้โดยตรง และระบบจะบันทึกว่าใครรับเคสนั้นไป

ส่วนประกอบหน้าที่
Schedulerรันงานตามเวลาที่กำหนด เช่น ทุก 5 นาที เพื่อ query ข้อมูลใหม่จาก HOSxP
Workerส่งข้อความ LINE ในเบื้องหลัง (async queue)
Web Appหน้าจัดการระบบ ดูสถิติ ตั้งค่า
MOPH NotifyGateway ส่ง LINE Flex Message ผ่าน MOPH API

2. การเข้าสู่ระบบ

รองรับการเข้าสู่ระบบ 2 แบบ

แบบ A — Username / Password (Admin)

1
เปิด URL ของระบบ → หน้า Login จะปรากฏขึ้น
2
กรอก Username และ Password
3
กรอก รหัส 2FA จาก Google Authenticator (ถ้าเปิดใช้งาน)

แบบ B — Provider Login (SSO)

1
กดปุ่ม "เข้าสู่ระบบด้วย Health ID"
2
ระบบจะ redirect ไปยังหน้า Health ID Portal ให้ยืนยันตัวตน
3
หลังยืนยันสำเร็จ ระบบดึงข้อมูล Profile มาสร้าง/อัปเดต Provider Profile อัตโนมัติ

4. Alert Cases — ติดตามเคส

หน้านี้แสดงรายการ Alert ทั้งหมดที่ระบบสร้างขึ้น พร้อมสถิติการรับเคส

สถิติด้านบน (Dashboard Stats)

Statความหมาย
Total Casesจำนวนเคสทั้งหมดในช่วงเวลาที่กรอง
รับแล้วเคสที่มีผู้กด Claim แล้ว
รอรับเคสเคสที่ยังไม่มีผู้รับ
Avg Claim Timeเวลาเฉลี่ยตั้งแต่สร้างเคสถึงรับเคส (นาที)
รับวันนี้จำนวนเคสที่รับแล้วในวันปัจจุบัน

ตัวกรอง

1
เลือก วันที่ (date_from / date_to) — default คือวันปัจจุบัน
2
กรอง ประเภท Alert (เช่น lab_critical, drug_critical)
3
กรอง สถานะ (OPEN / CLAIMED)
4
กดปุ่ม Export CSV / XLSX เพื่อดาวน์โหลดข้อมูล
สถิติ Dashboard ด้านบนจะนับตามเงื่อนไขที่กรองอยู่เสมอ ไม่ใช่ข้อมูลทั้งหมด

5. Manual Claim

กรณีที่ปุ่มรับเคสใน LINE ไม่ทำงาน หรือลิงก์หมดอายุ เจ้าหน้าที่สามารถเข้ามารับเคสจากหน้านี้โดยตรง

1
เลือกช่วง วันที่ ที่ต้องการ (default = วันปัจจุบัน)
2
ดูรายการเคสในตาราง — สถานะ OPEN คือยังไม่มีผู้รับ
3
กดปุ่ม "รับเคส" ในแถวที่ต้องการ → ระบบจะพาไปหน้ายืนยัน
หน้านี้ต้องเข้าสู่ระบบก่อน ต่างจากลิงก์ใน LINE ที่ไม่ต้อง login

7. Alert Type Configs

กำหนดประเภทการแจ้งเตือนต่างๆ เช่น lab_critical, drug_critical แต่ละประเภทมีการตั้งค่าเป็นของตัวเอง

Fieldความหมายตัวอย่าง
Type Codeรหัสอ้างอิงในระบบlab_critical
Display Nameชื่อที่แสดงในหน้าจอผลแล็บวิกฤต
Bubble Titleชื่อหัว Bubble LINE รองรับ {field}🔴 {lab_items_name}
Bubble Colorสีหัว Bubble (HEX)#DC2626
Required Fieldsชื่อ field ที่ต้องมีในข้อมูล (JSON array)["hn","lab_items_name"]
Key FieldsField ที่ใช้ dedup ไม่ส่งซ้ำ (JSON array)["hn","lab_order_number"]
หาก Bubble Title ใส่ {field} เช่น {lab_items_name} ระบบจะดึงค่าจากข้อมูลแถวนั้นมาแสดงแทนให้อัตโนมัติ

8. Claim Notify Settings

ตั้งค่าข้อความที่จะส่งกลับไปยัง LINE หลังจากมีผู้รับเคสแล้ว

Fieldความหมาย

9. Notify Rooms

จัดการ ห้อง LINE ที่ใช้รับข้อความแจ้งเตือน แต่ละห้องมี Client Key / Secret Key เป็นของตัวเอง (Multi-tenant)

1

2

4
กด "บันทึก"
ห้องที่ตั้งค่าไว้จะปรากฏใน dropdown เมื่อทดสอบส่งข้อความในหน้า Notify Test

10. Schedules

จัดการงานที่รันอัตโนมัติตามเวลาที่กำหนด เช่น ดึงข้อมูลแล็บจาก HOSxP ทุก 5 นาที

Fieldความหมาย
Job Nameชื่อของงาน
Alert Typeประเภทแจ้งเตือนที่งานนี้รับผิดชอบ
QuerySQL Query ที่ใช้ดึงข้อมูล
TemplateFlex Template ที่ใช้ส่ง LINE
Notify Roomห้อง LINE ที่ส่งข้อมูลไป
Cron Expressionเวลาที่รัน เช่น */5 * * * * = ทุก 5 นาที
Activeเปิด/ปิดงานนี้
Cron Expression ตัวอย่าง: */5 * * * * ทุก 5 นาที | 0 8 * * * ทุกวัน 8:00 น. | 0 */2 * * * ทุก 2 ชั่วโมง

11. Scheduler Monitor

หน้ามอนิเตอร์ดูสถานะของ Scheduler แบบ Real-time โดยไม่ต้องเข้า server

ส่วนข้อมูลที่แสดง
Stats Barจำนวน Jobs ทั้งหมด / Active / Inactive / Total Runs
Job Cardsแต่ละ job แสดง Cron Expression, Next Run, Last Run, สถานะ Success/Failed
Log Tableประวัติการรัน พร้อมสถานะและ error message (ถ้ามี)
ปุ่ม Run Nowรัน Job ทันทีโดยไม่ต้องรอเวลา
หาก Job แสดงสถานะ Failed ให้กดดู error detail ใน Log Table เพื่อวิเคราะห์สาเหตุ
{# 12 ── Queries ── #}

12. Approved Queries

จัดการ SQL Query ที่ได้รับอนุมัติให้ใช้ในระบบ เพื่อดึงข้อมูลจาก HOSxP

1
กดปุ่ม "เพิ่ม Query ใหม่"
2
กรอก ชื่อ Query และ SQL Statement (รองรับเฉพาะ SELECT)
3
กด "Preview" เพื่อทดสอบ Query กับ HOSxP จริง ดูตัวอย่างผลลัพธ์
4
กด "บันทึก" เพื่อเพิ่มเข้า Approved List
ระบบอนุญาตเฉพาะ SELECT เท่านั้น ไม่อนุญาต INSERT / UPDATE / DELETE / DROP
{# 13 ── Templates ── #}
🎨

13. Templates

จัดการ Flex Message Template สำหรับส่ง LINE แต่ละ Template เป็น JSON ที่ออกแบบรูปแบบข้อความไว้

รองรับ variable substitution โดยใช้ {field_name} ในข้อความ เช่น {patient_name}, {hn} ระบบจะแทนค่าจากข้อมูลแถวที่ส่งมาโดยอัตโนมัติ

กดปุ่ม "Dynamic Flex Builder" เพื่อสร้าง Template แบบ visual โดยไม่ต้องเขียน JSON เอง

{# 16 ── Profiles ── #}

16. Provider Profiles

ข้อมูลโปรไฟล์ของผู้ใช้ที่ login ผ่าน Health ID (Provider Login) ระบบดึงข้อมูลมาจาก Provider API อัตโนมัติ

ฟังก์ชันรายละเอียด
ดูรายละเอียดกดชื่อเพื่อดู Profile เต็ม รวมถึง Raw JSON จาก Provider
แก้ไขข้อมูลแก้ไข ชื่อ ตำแหน่ง หน่วยงาน เบอร์โทร อีเมล
Sync จาก Provider JSONดึงข้อมูลจาก Raw JSON ที่ได้จาก Provider มาเติมทุก field ใหม่
ประวัติการเปลี่ยนแปลงดูว่า field ไหนถูกแก้ไขเมื่อใด โดยใคร
Exportส่งออกรายการ Profiles เป็น CSV / XLSX

17. Logs

ดูประวัติการทำงานของระบบ แบ่งเป็น 3 แท็บ

แท็บบันทึกอะไร
Access Logsการ Login / Logout, การเข้าถึงหน้าต่างๆ, IP Address
Send Logsประวัติการส่ง LINE ทุกครั้ง สถานะ Success/Failed, Retry Count
Deliveryสถานะ Delivery ที่ได้รับกลับจาก MOPH Notify Gateway

มีตัวกรองวันที่, ค้นหาคำ, กรองสถานะ, กรอง Action และปุ่ม Export ทุกแท็บ
