1. ภาพรวมระบบ

BK-Moph Notify คือระบบแจ้งเตือนผลวิกฤต (Critical Alert) สำหรับโรงพยาบาล โดยดึงข้อมูลจากฐานข้อมูล HOSxP แล้วส่งข้อความแจ้งเตือนผ่าน LINE ไปยังกลุ่มเจ้าหน้าที่ที่เกี่ยวข้อง เช่น ผลแล็บวิกฤต (Lab Critical) หรือยาวิกฤต (Drug Critical)

เจ้าหน้าที่รับทราบข้อมูลแล้วสามารถ กดรับเคส (Claim) ผ่านปุ่มในข้อความ LINE ได้โดยตรง และระบบจะบันทึกว่าใครรับเคสนั้นไป

ส่วนประกอบหน้าที่
Schedulerรันงานตามเวลาที่กำหนด เช่น ทุก 5 นาที เพื่อ query ข้อมูลใหม่จาก HOSxP
Workerส่งข้อความ LINE ในเบื้องหลัง (async queue)
Web Appหน้าจัดการระบบ ดูสถิติ ตั้งค่า
MOPH NotifyGateway ส่ง LINE Flex Message ผ่าน MOPH API

4. Alert Cases — ติดตามเคส

หน้านี้แสดงรายการ Alert ทั้งหมดที่ระบบสร้างขึ้น พร้อมสถิติการรับเคส

สถิติด้านบน (Dashboard Stats)

Statความหมาย
Total Casesจำนวนเคสทั้งหมดในช่วงเวลาที่กรอง
รับแล้วเคสที่มีผู้กด Claim แล้ว
รอรับเคสเคสที่ยังไม่มีผู้รับ
Avg Claim Timeเวลาเฉลี่ยตั้งแต่สร้างเคสถึงรับเคส (นาที)
รับวันนี้จำนวนเคสที่รับแล้วในวันปัจจุบัน

ตัวกรอง

1
เลือก วันที่ (date_from / date_to) — default คือวันปัจจุบัน
2
กรอง ประเภท Alert (เช่น lab_critical, drug_critical)
3
กรอง สถานะ (OPEN / CLAIMED)
4
กดปุ่ม Export CSV / XLSX เพื่อดาวน์โหลดข้อมูล
สถิติ Dashboard ด้านบนจะนับตามเงื่อนไขที่กรองอยู่เสมอ ไม่ใช่ข้อมูลทั้งหมด

5. Manual Claim

1

2
ดูรายการเคสในตาราง — สถานะ OPEN คือยังไม่มีผู้รับ
3
กดปุ่ม "รับเคส" ในแถวที่ต้องการ → ระบบจะพาไปหน้ายืนยัน
หน้านี้ต้องเข้าสู่ระบบก่อน ต่างจากลิงก์ใน LINE ที่ไม่ต้อง login

7. Alert Type Configs — ขั้นตอนสร้างการแจ้งเตือนใหม่ตั้งแต่ต้น

การสร้างการแจ้งเตือนประเภทใหม่ (เช่น ยาวิกฤต, ยาซ้ำซ้อน, ผลแล็บวิกฤต) ต้องทำตามลำดับ 4 ขั้นตอน ดังนี้

🔍
1. สร้าง Query
Approved Queries
⚙️
2. สร้าง Config
Alert Type Configs
🎨
3. สร้าง Template
Claim Alert Builder
⏱️
4. ตั้ง Schedule
Schedules
{# Step 1 #}
ขั้นตอนที่ 1 — สร้าง Approved Query

ไปที่เมนู Approved Queries → กดปุ่ม "เพิ่ม Query ใหม่"

1
กรอก ชื่อ Query ที่สื่อความหมาย เช่น drug_critical_egfr
2
เขียน SQL SELECT ที่ดึงข้อมูลจาก HOSxP — ต้องมี column ที่จะใช้ใน Alert ทุกตัว เช่น hn, ptname, drug_name, egfr
3
กด "Preview" เพื่อดูผลลัพธ์จริงจาก HOSxP — ตรวจสอบชื่อ column ที่ได้ออกมา เพราะจะต้องใช้ในขั้นตอนที่ 2
4
กด "บันทึก"
จดชื่อ column ที่ได้จาก Preview ไว้ก่อน เช่น hn, ptname, drug_name, egfr, result, vstdate เพราะต้องใช้ในขั้นตอนถัดไป
{# Step 2 #}
ขั้นตอนที่ 2 — สร้าง Alert Type Config

ไปที่เมนู Alert Type Configs → กรอกฟอร์มด้านซ้าย

Fieldคืออะไรตัวอย่าง
Type Code รหัสอ้างอิงในระบบ — ห้ามซ้ำ ห้ามมีช่องว่าง drug_critical_egfr
Display Name ชื่อที่แสดงในหน้าจอ ยา Metformin เกินขนาด eGFR ต่ำ
Bubble Title ข้อความหัว bubble — รองรับ {column} เพื่อดึงค่าจากข้อมูลจริง 💊 {drug_name}
Bubble Title Color สีหัว bubble (HEX code) #DC2626 (แดง)
Required Fields JSON array — ชื่อ column ที่ต้องมีค่าในแต่ละ row ถ้าไม่มีจะข้ามแถวนั้น ["hn","ptname","drug_name","egfr"]
Key Fields JSON array — column ที่ใช้สร้าง key ป้องกันส่งซ้ำ (dedup) ควรรวม column ที่ระบุตัวเคสได้แน่นอน ["hn","drug_name","vstdate"]
Field Map JSON object — map ชื่อ standard ของระบบ → ชื่อ column จริงใน query ใช้สร้าง header ของ bubble และ claim notify template 
{"patient_hn":"hn",
"patient_name":"ptname",
"department":"cur_dep",
"doctor":"doctor_name"}
Display Lines JSON array — กำหนดบรรทัดเนื้อหาใน bubble ใช้ {column} แทนค่าจริง รองรับ size, weight, color, bold 
[
{"text":"eGFR: {egfr}"},
{"text":"⚠️ {result}",
 "weight":"bold",
 "color":"#DC2626"}
]
Claim Notify Template ข้อความส่งกลับ LINE หลังรับเคส — ว่างได้ จะใช้ค่า default รองรับตัวแปร {patient_name}, {patient_hn}, {claimed_by}, {claimed_at} ✅ รับเคส {patient_name} แล้ว โดย {claimed_by}
โครงสร้าง bubble ที่ได้:
[ Bubble Title ] ← จาก bubble_title (รองรับ {column})
[ ชื่อผู้ป่วย ] ← จาก field_map → patient_name
[ HN | แผนก ] ← จาก field_map → patient_hn + department
──────────────
[ Display Lines... ] ← แสดงตาม display_lines ที่กำหนด
[ แพทย์ผู้สั่ง ] ← จาก field_map → doctor
[ สถานะเคส ]
[ ปุ่มรับเคส ]
ข้อสำคัญ: Field Map ต้องเป็น JSON object {} เท่านั้น | Display Lines ต้องเป็น JSON array [] เท่านั้น — อย่าสลับกัน
{# Step 3 #}
ขั้นตอนที่ 3 — สร้าง Claim Alert Template

ไปที่เมนู Alert Type Configs → กดปุ่ม "สร้าง Claim Alert Template" (มุมขวาบน)

1
กรอก ชื่อ Template เช่น แจ้งเตือนการใช้ metformin เกินขนาด
2
เลือก Alert Type จาก dropdown — เลือก Config ที่สร้างในขั้นตอนที่ 2
3
กด "บันทึก" — ระบบจะสร้าง Template ชนิด claim_alert พร้อม content {"alert_type_code": "..."} ให้อัตโนมัติ
Template ชนิด claim_alert ไม่ต้องเขียน Flex JSON เอง — ระบบจะ render bubble จาก Alert Type Config โดยอัตโนมัติ
{# Step 4 #}
ขั้นตอนที่ 4 — ตั้ง Schedule

ไปที่เมนู Schedules → กดปุ่ม "เพิ่ม Schedule ใหม่"

1
กรอก Job Name เช่น drug_critical_egfr_notify
2
เลือก Template = Template ที่สร้างในขั้นตอนที่ 3
3
เลือก Query = Query ที่สร้างในขั้นตอนที่ 1
4
เลือก Notify Room = ห้อง LINE ที่จะรับการแจ้งเตือน
5
ตั้ง Cron Expression เช่น */15 * * * * = ทุก 15 นาที
6
กด "บันทึก" แล้วไปที่ Scheduler Monitor → กด "Run Now" เพื่อทดสอบทันที
Cron Expressionความหมาย
*/5 * * * *ทุก 5 นาที
*/15 * * * *ทุก 15 นาที
0 * * * *ทุกชั่วโมง (นาทีที่ 0)
0 8 * * *ทุกวัน เวลา 08:00 น.
0 8,12,16 * * *วันละ 3 ครั้ง 08:00, 12:00, 16:00 น.
ถ้าข้อมูลเคสเดิมถูกส่งแล้ว (Key Fields ตรงกัน) ระบบจะ ไม่ส่งซ้ำ — เฉพาะเคสใหม่เท่านั้นที่จะถูกส่ง

8. Claim Notify — ข้อความตอบกลับหลังรับเคส

เมื่อมีผู้กดปุ่ม "รับเคส" ในห้อง LINE ระบบจะส่งข้อความยืนยันกลับไปยังห้องนั้นทันที คุณสามารถกำหนด รูปแบบ และ เนื้อหา ของข้อความนี้ได้ใน Alert Type Config → ส่วน Claim Notify

ตั้งค่าได้ที่ Alert Type Configs → เลือก Config ที่ต้องการ → กด Edit → เลื่อนลงไปที่ส่วน "ประเภทข้อความตอบกลับ"

ประเภทข้อความ (Claim Notify Type)

ประเภทลักษณะเหมาะสำหรับ
💬 Text ข้อความธรรมดา รองรับตัวแปร {field} และ emoji ข้อความสั้น กระชับ อ่านง่าย
🎨 Flex LINE Flex Message (JSON bubble/carousel) — การ์ดสวยงาม จัดวาง layout ได้ ต้องการ UI ที่สวยงาม แสดงหลาย field อย่างเป็นระเบียบ

ตัวแปรที่ใช้ได้ใน Template

ใช้ {ชื่อตัวแปร} ในข้อความหรือ JSON — ระบบจะแทนค่าอัตโนมัติ

ตัวแปรข้อมูลที่แทน
{patient_name}ชื่อ-นามสกุลผู้ป่วย
{patient_hn}HN ผู้ป่วย
{item_name}ชื่อรายการ (ยา/LAB) จาก field_map → item_name
{item_value}ค่า/ผล จาก field_map → item_value
{department}แผนกผู้ป่วย
{claimed_by}ชื่อผู้รับเคส (username)
{claimed_at}เวลาที่รับเคส (เวลาไทย)
{alert_type}type_code ของ Alert Type Config
{ชื่อ column ใดๆ}ทุก column จาก Query ที่ดึงข้อมูล (source_row_json)
ถ้า field_map กำหนดว่า item_name → drug_name คุณยังสามารถใช้ {drug_name} โดยตรงได้ด้วย เพราะระบบโหลดทุก column จาก Query

💬 รูปแบบ Text — ตัวอย่าง

ใส่ข้อความธรรมดาในช่อง Claim Notify Template (ถ้าเว้นว่างจะใช้ค่า default)

{% raw %}
✅ รับเคสเรียบร้อย

👤 ผู้ป่วย: {patient_name}
🏷 HN: {patient_hn}
💊 ยา: {item_name}  ขนาด: {item_value}
🏥 แผนก: {department}
⚠️ ผล EGFR: {egfr}
✋ ผู้รับเคส: {claimed_by}
🕐 เวลา: {claimed_at}
{% endraw %}
ค่า {egfr} จะดึงจาก column egfr ใน query โดยตรง แม้ไม่ได้อยู่ใน field_map

🎨 รูปแบบ Flex — ตัวอย่าง

เลือก Flex Message แล้วใส่ JSON ของ contents (bubble หรือ carousel) ในช่อง Template — ใช้ {field} ใน text ได้ทุกที่

{% raw %}
{
  "type": "bubble",
  "body": {
    "type": "box",
    "layout": "vertical",
    "contents": [
      {
        "type": "text",
        "text": "✅ รับเคสเรียบร้อย",
        "weight": "bold",
        "color": "#15803d",
        "size": "md"
      },
      { "type": "separator", "margin": "md" },
      {
        "type": "box",
        "layout": "vertical",
        "margin": "md",
        "spacing": "sm",
        "contents": [
          {"type":"text","text":"👤 {patient_name}","size":"sm"},
          {"type":"text","text":"🏷 HN: {patient_hn}","size":"sm"},
          {"type":"text","text":"💊 {item_name}  ค่า: {item_value}","size":"sm","color":"#dc2626","weight":"bold"},
          {"type":"text","text":"🏥 {department}","size":"sm"},
          {"type":"separator","margin":"sm"},
          {"type":"text","text":"✋ {claimed_by}  🕐 {claimed_at}","size":"xs","color":"#64748b"}
        ]
      }
    ]
  }
}
{% endraw %}
ข้อควรระวัง Flex:
  • ต้องเป็น JSON ที่ถูกต้อง — ผิดพลาดระบบจะ fallback เป็น Text แทน
  • ใส่เฉพาะ contents (bubble/carousel object) — ไม่ต้องใส่ type: "flex" ชั้นนอก
  • ทดสอบ Flex JSON ได้ที่ LINE Flex Message Simulator

ขั้นตอนตั้งค่า

1
ไปที่ Alert Type Configs → กด Edit บน config ที่ต้องการ
2
เลือก ประเภทข้อความตอบกลับ: 💬 Text หรือ 🎨 Flex
3
พิมพ์ template ในช่อง Claim Notify Template — ถ้าเว้นว่างไว้จะใช้ข้อความ default ของระบบ
4
กด Update — ข้อความที่ตั้งค่าจะมีผลทันทีในการรับเคสครั้งถัดไป
ไม่ต้อง restart server — การตั้งค่าโหลดใหม่ทุกครั้งที่มีการรับเคส
