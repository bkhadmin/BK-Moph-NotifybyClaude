1. ภาพรวมและวัตถุประสงค์
Executive Summary & Objectives

ระบบ BK-Moph-Notify เป็นแพลตฟอร์มสำหรับบริหารจัดการการส่งข้อความแจ้งเตือนผู้ป่วยวิกฤตในโรงพยาบาลบางระกำ โดยดึงข้อมูลจากระบบ HOSxP แบบ real-time แล้วส่งข้อความไปยังเจ้าหน้าที่ที่เกี่ยวข้องผ่านช่องทาง MOPH Notify และ Telegram พร้อมระบบ "รับเคส" เพื่อติดตามการดูแลผู้ป่วย

🔬
LAB วิกฤต
แจ้งเตือนค่า Lab ที่ผิดปกติวิกฤต เช่น K, Na, Glucose ผิดปกติรุนแรง ส่งตรงถึงแพทย์/พยาบาลที่ดูแล
lab_critical
💊
ยาวิกฤต (EGFR)
แจ้งเตือนผู้ป่วยที่ได้รับยา Metformin แต่มีค่า eGFR ต่ำ เสี่ยงต่อภาวะ Lactic Acidosis
drug_critical_egfr
⚠️
ผู้ป่วย MDR
แจ้งเตือนผู้ป่วยที่มีเชื้อดื้อยา (Multi-Drug Resistant) เพื่อให้เจ้าหน้าที่ดำเนินมาตรการป้องกันการแพร่กระจาย
mdr_critical

ประโยชน์ที่ได้รับ

⚡
แจ้งเตือนรวดเร็ว
ตรวจสอบข้อมูลทุก 5 นาที ลดระยะเวลาการรู้ข้อมูลผู้ป่วยวิกฤตจากชั่วโมงเหลือเพียงนาที
✅
ติดตามการรับเคส
เจ้าหน้าที่กดรับเคสผ่าน Link ใน LINE ทำให้ทราบว่ามีผู้รับผิดชอบดูแลผู้ป่วยแล้ว
📊
รายงานและสถิติ
ดู Claim Rate รายวัน export ข้อมูลสำหรับการประเมินคุณภาพการดูแลผู้ป่วย
🔀
ส่งตรงห้องที่เกี่ยวข้อง
ระบบ Alertroom อ่านรหัสห้องจาก Query แล้วส่งข้อความตรงไปยังห้องที่รับผิดชอบผู้ป่วย
📱
รองรับมือถือ
หน้า Claim รองรับมือถือครบถ้วน เจ้าหน้าที่กดรับเคสได้ทันทีจาก LINE Notification
🔒
ปลอดภัย
Claim Link มี HMAC signature และ expiry 1 ชั่วโมง ป้องกันการปลอมแปลง

4.4 ระบบ Alertroom (ส่งตามห้องอัตโนมัติ)

Query ที่มี field alertroom

SELECT
  hn, vn, ptname,
  plain_text,
  main_dep AS alertroom,
  main_department
FROM mdr_alert_view
WHERE DATE(report_date) = CURDATE()

การทำงานของระบบ

1. อ่าน alertroom จากทุก row เช่น "061,062"

2. ค้นหา notify_rooms ที่ room_code ตรงกัน

3. ส่งข้อความไปทุกห้องที่พบ

4. บันทึก notify_room_id ใน alert_case เพื่อ claim notify กลับถูกห้อง

⚠️ ถ้าไม่พบห้อง → fallback ไป default room ใน .env

6. ช่องทางการแจ้งเตือน
MOPH Notify & Telegram Integration
📨
MOPH Notify

ส่งผ่าน POST /api/notify/send

• client-key และ secret-key ใน header

• รองรับ messages[] — Text และ Flex

• ส่งเข้าตรง LINE Group ที่ลงทะเบียนกับ MOPH

channel_type: moph_notify
✈️
Telegram Bot

ส่งผ่าน Telegram Bot API

• Bot Token (client_key) + Chat ID (secret_key)

• Flex JSON แปลงเป็น plain text อัตโนมัติ

• แนบ Claim URL ท้ายข้อความ

channel_type: telegram
💡
สามารถตั้งค่าได้หลายห้องต่อประเภท — เช่น ห้อง LAB (MOPH Notify) + ห้อง MDR (Telegram) ทำงานคู่ขนานกันได้ โดยกำหนดผ่านเมนู Notify Rooms
